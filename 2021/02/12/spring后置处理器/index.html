spring后置处理器

        本章主要分析beanFactory的后置处理器，以及bean的后置处置器。即：BeanFactoryPostProcessor和BeanPostProcessor。关于BeanPostProcessor，本节主要讲注册，具体调用，会在bean的实例化之后写到。

什么是后置处理器？

首先，我先来说一下我对于后置处理器的理解。

  • beanFactoryPostProcessor

        上一章中已经分析过，在beanFactory的填充过程中，会封装很多beanDefinition到map容器中，但是这些bd都是要提前配置的，比如在spring.xml中配置bean，那么我们有没有别的办法对容器中的beanDefinition进行增删改查呢？可以的，beanFactoryPostProcessor就可以做这个工作。比如：实际上现在的springboot项目不会有spring.xml文件的，那么bean的收集工作并不会在obtainFreshBeanFactory中完成，而且交给一个ConfigurationClassPostProcessor来做的，它会完成@Import、@Component、@Bean、@Condition、@PropertySource等注解的处理，后面有空我会着重写到。

  • beanPostProcessor

        简单来讲，我认为就是各种埋点，你可以实现很多beanPostProessor，bean在实例化的不同时期，比如，注解的收集、popularBean、initializeBean过程中，都会调用到这些埋点，来做一些事情，还有，aop的入口其实也是一个beanPostProcessor。

spring 初始化核心流程

  • ├─ refresh Spring 初始化入口
  • │ ├─ prepareRefresh ① 初始化一些参数，设置启动时间和active标志
  • │ ├─ obtainFreshBeanFactory ② 创建 BeanFactory
  • │ ├─ prepareBeanFactory ③ 设置 BeanFactory 的基本属性
  • │ ├─ postProcessBeanFactory ④ 允许注册特殊的BPP
  • │ ├─ invokeBeanFactoryPostProcessors 本节主要跟踪的源码流程
  • │ ├─ registerBeanPostProcessors 本节主要跟踪的源码流程
  • │ ├─ initMessageSource ⑦ 初始化资源文件，如国际化文件的处理等
  • │ ├─ initApplicationEventMulticaster ⑧ 初始化事件管理类
  • │ ├─ onRefresh ⑨ 给子类扩展初始化其他Bean，springboot 中用来做内嵌 tomcat 启动
  • │ ├─ registerListeners ⑩ 在所有bean中查找listeners，然后注册到广播器中
  • │ ├─ finishBeanFactoryInitialization ⑪ 完成bean实例化
  • │ └─ finishRefresh ⑫ 完成refresh，发布相应事件

首先对实现了①、②接口的实现类优先实例化，并直接调用实现类中的两个实现方法。

  • ①BeanDefinitionRegistryPostProcessor

    • 调用实现类中的postProcessBeanDefinitionRegistry，完成对容器中的beanDefinition进行增删改查。

  • ②BeanFactoryPostProcessor

    • 调用实现类中postProcessBeanFactory，拿到beanFactory做一些操作，比如提前getBean("xxx")某个bd。
